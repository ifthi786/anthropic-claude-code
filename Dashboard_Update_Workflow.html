## Task
Rebuild Critical_Packages_by_Project.xlsx from the updated Provisional_Sum_Nomination_Tracker.xlsx.

## Source & Projects
- Source: Provisional_Sum_Nomination_Tracker.xlsx
- Project sheets (exact names): Mamsha Palm | Grove 6&7 | Fahid TA1-2
- If a listed sheet is missing or renamed, stop and ask — don't guess.

## Sheet Structure (critical — do not deviate)
- Read with openpyxl, data_only=True (cached formula values). Do NOT use read_only=True.
- Data rows start at row 11.
- A valid package row = column A contains a numeric item number (1..N).
  Rows with blank A are section headers (text in col B) or Total rows — skip them.
- Column map: A = Item No. | B = Package No. (may be blank or combined, e.g. "PS-25 & PS-27") |
  C = BOQ group label (MERGED cells — never use as package name) |
  D = Package Name (true sub-bill name, e.g. "Structural Steel", "Aluminium Doors,Glass Doors,
  Curtain Walls, Glass Balustrade, External Louver") | E = Package Manager |
  AC (col 29) = Variance from Today → Days Remaining | BI (col 61) = Current Stage
- Package Name = column D, whitespace-normalized. If D is blank, fall back to column C
  (e.g. Mamsha "Attic Stock").

## Extraction & Filter
1. Capture EVERY column-A-numbered row per sheet.
2. STAGE FILTER: include only packages that have NOT reached Award Rec. Approval —
   i.e. include stages 1–8 only; exclude any Current Stage starting with "9.", "10." or "11."
3. DAYS FILTER: include only packages with Days Remaining ≤ 45 (including all negatives).
   Packages above 45 days are excluded entirely — no GREEN tier in this report.
4. Exclude non-numeric Days Remaining from the report, but FLAG any such row at stage 1–8.
5. No top-10 cap — capture ALL packages meeting both filters, even if a project has
   0 or 20+ qualifying packages. If a project has none, still create its sheet with a
   note: "No packages within 45 days at stage 1–8."
6. Sort: Days Remaining ascending → earlier stage number first (tie-break) → name.

## Criticality Tiers (3 levels)
- OVERDUE:   Days Remaining < 0          → fill C00000, white bold text
- CRITICAL:  Days Remaining 0–15         → fill ED7D31, white bold text
- ATTENTION: Days Remaining 16–45        → fill FFC000, black bold text
Add a "Criticality" column showing the tier label. Apply the tier fill to both the
Criticality cell and the Days Remaining cell. Enable Excel AutoFilter on the header
row of every sheet so the report can be filtered by Criticality tier.

## Mandatory Audit (report with delivery)
- Per project: items found vs max column-A number — must match with no gaps (e.g. 37/37).
- Counts: total items → after stage filter → after 45-day filter (per project).
- List packages excluded by the stage filter at stage 9 (Award Rec. Approval) specifically.
- Any stage 1–8 package with non-numeric Days Remaining.
- Tier distribution: Overdue / Critical / Attention counts.

## Output Workbook
Sheets: Summary (all qualifying packages, globally sorted by Days Remaining) +
Mamsha Palm + Grove 6&7 + Fahid TA1-2.
Per-sheet layout:
- Row 1: title, merged, fill 1F4E79, white bold 14pt Arial
- Row 2: "Data as of: DD Mon YYYY" — static generation date (not live TODAY())
- Row 4: headers — Rank | Project (Summary only) | Package No. | Package Name |
  Package Manager | Current Stage | Days Remaining | Criticality
  (fill 2E75B6, white bold, AutoFilter ON)
- Data from row 5: Arial 10, thin gray borders, wrap text on Name/Stage,
  center Rank/Package No./Days/Criticality, freeze panes below header, gridlines off,
  tab colors (Summary 1F4E79, projects 2E75B6), widths ≈ 6/14/14/56/16/23/14/14
- Criticality legend (3 tiers with day ranges) below the Summary table.
